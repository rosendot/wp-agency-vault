# Accessibility Improve

Reads the `a11y-report.md` generated by the Accessibility Analyzer agent and applies the fixes.

## Usage
`/a11y-improve <template-slug>`

## Prerequisites
The Accessibility Analyzer agent must have been run first. This skill reads `a11y-report.md` from the template directory. If the report does not exist, tell the user:

> Run the Accessibility Analyzer agent on this template first. It will generate the `a11y-report.md` that this skill needs.

## Steps

1. Read `templates/$ARGUMENTS/a11y-report.md` (or the path the user specifies).
2. Parse all **Critical** and **Warning** issues from the report. Each issue has:
   - File path
   - Line number(s)
   - WCAG criterion
   - Problem description
   - Fix instruction
3. For each issue, starting with Critical then Warnings:
   - Read the referenced file
   - Apply the fix described in the report
   - Verify the fix doesn't break surrounding code or layout
4. After all Critical and Warning fixes are applied, list the **Suggestions** and ask the user which ones they want applied.
5. After all chosen fixes are applied, delete `a11y-report.md` — it is now stale.
6. Summarize what was changed:
   - Files modified
   - Issues fixed (by severity and WCAG criterion)
   - Manual verification items still outstanding
   - Suggestions skipped or applied

## Fix Patterns

### Skip-nav link
Add as first child of `<body>`:
```php
<a class="skip-link screen-reader-text" href="#main-content">
	<?php esc_html_e( 'Skip to content', 'template-slug' ); ?>
</a>
```
Add corresponding CSS:
```css
.skip-link {
	position: absolute;
	left: -9999px;
	top: auto;
	width: 1px;
	height: 1px;
	overflow: hidden;
}
.skip-link:focus {
	position: fixed;
	top: 0;
	left: 0;
	width: auto;
	height: auto;
	padding: 1rem 2rem;
	background: #000;
	color: #fff;
	z-index: 100000;
	font-size: 1rem;
}
```
Add `id="main-content"` to the `<main>` element.

### Missing form labels
```html
<!-- Before -->
<input type="email" name="email" placeholder="Your email">
<!-- After -->
<label for="email-field">Email address</label>
<input type="email" name="email" id="email-field" placeholder="Your email">
```

### Missing ARIA on navigation
```php
<!-- Before -->
<nav>
<!-- After -->
<nav aria-label="<?php esc_attr_e( 'Primary navigation', 'template-slug' ); ?>">
```

### Focus styles
Add to template CSS if missing:
```css
:focus-visible {
	outline: 2px solid var(--color-primary);
	outline-offset: 2px;
}
```

### Div buttons → real buttons
```html
<!-- Before -->
<div class="btn" onclick="doThing()">Click me</div>
<!-- After -->
<button class="btn" type="button">Click me</button>
```

### Missing alt attributes
```php
<!-- Before -->
<img src="<?php echo esc_url($src); ?>">
<!-- After — informative image -->
<img src="<?php echo esc_url($src); ?>" alt="<?php echo esc_attr($description); ?>">
<!-- After — decorative image -->
<img src="<?php echo esc_url($src); ?>" alt="">
```

### Keyboard trap in modals
Ensure modals have:
```js
// Close on Escape
document.addEventListener('keydown', function(e) {
	if (e.key === 'Escape') closeModal();
});
// Trap focus within modal while open
// Return focus to trigger element on close
```

### Missing language attributes
```php
<!-- Before -->
<html>
<!-- After -->
<html <?php language_attributes(); ?>>
```

## Rules
- Only fix issues listed in `a11y-report.md` — do not go looking for additional issues
- Apply Critical and Warning fixes without asking. Ask before applying Suggestions.
- All output must remain properly escaped (esc_html, esc_attr, esc_url)
- Preserve existing functionality — accessibility fixes should be additive, not destructive
- Delete `a11y-report.md` when done — a stale report is worse than no report
- If a fix instruction is ambiguous or could break layout/functionality, flag it to the user instead of guessing
- Items in "Manual verification needed" cannot be fixed automatically — remind the user to test those in a browser
