# SEO Improve

Reads the `seo-report.md` generated by the SEO Analyzer agent and applies the fixes.

## Usage
`/seo-improve <template-slug>`

## Prerequisites
The SEO Analyzer agent must have been run first. This skill reads `seo-report.md` from the template directory. If the report does not exist, tell the user to run the SEO Analyzer agent first:

> Run the SEO Analyzer agent on this template first. It will generate the `seo-report.md` that this skill needs.

## Steps

1. Read `templates/$ARGUMENTS/seo-report.md` (or the path the user specifies).
2. Parse all **Critical** and **Warning** issues from the report. Each issue has:
   - File path
   - Line number(s)
   - Problem description
   - Fix instruction
3. For each issue, starting with Critical then Warnings:
   - Read the referenced file
   - Apply the fix described in the report
   - Verify the fix doesn't break surrounding code
4. After all Critical and Warning fixes are applied, list the **Suggestions** and ask the user which ones (if any) they want applied.
5. After all chosen fixes are applied, delete `seo-report.md` — it is now stale.
6. Summarize what was changed:
   - Files modified
   - Issues fixed (by severity)
   - Suggestions skipped or applied

## Fix Patterns

Common fixes this skill applies:

### Missing alt text
```php
<!-- Before -->
<img src="<?php echo esc_url($image); ?>">
<!-- After -->
<img src="<?php echo esc_url($image); ?>" alt="<?php echo esc_attr($alt_text); ?>">
```

### Missing wp_head / wp_footer
Add `<?php wp_head(); ?>` before `</head>` and `<?php wp_footer(); ?>` before `</body>`.

### Duplicate h1 tags
Keep the most semantically appropriate `<h1>`, demote others to `<h2>`.

### Heading hierarchy gaps
Adjust skipped heading levels (e.g., h1 → h4 becomes h1 → h2).

### Missing lazy loading
Add `loading="lazy"` to images below the fold. Do NOT add to hero/above-fold images.

### Missing width/height on images
Add `width` and `height` attributes to prevent CLS.

### External links missing rel
```html
<!-- Before -->
<a href="https://external.com" target="_blank">
<!-- After -->
<a href="https://external.com" target="_blank" rel="noopener noreferrer">
```

### Render-blocking scripts
Move scripts to footer or add `defer`:
```php
wp_enqueue_script('handle', $src, array(), $ver, true); // true = in footer
```

### Hardcoded meta conflicting with Rank Math
Remove hardcoded `<meta name="description">`, `<meta name="robots">`, or Open Graph tags that conflict with Rank Math output. These should be managed by the plugin, not the template.

## Rules
- Only fix issues listed in `seo-report.md` — do not go looking for additional issues
- Apply Critical fixes without asking. Apply Warning fixes without asking. Ask before applying Suggestions.
- Never hardcode meta tags that Rank Math should manage (description, robots, OG tags, schema)
- All output must remain properly escaped (esc_html, esc_attr, esc_url)
- Delete `seo-report.md` when done — a stale report is worse than no report
- If a fix instruction in the report is ambiguous or could break functionality, flag it to the user instead of guessing
